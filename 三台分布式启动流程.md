# 三台分布式启动流程（Tailscale）

> **用途**：三人三机联调、答辩演示时的操作清单。  
> **网络**：Tailscale 同一 tailnet，全程使用 `100.x.x.x` 地址。  
> **更新**：2026-06-22

---

## 1. 分工一览

| 人员 | 机器职责 | 关键进程 |
|------|----------|----------|
| **Person A** | 基础设施 + 调度 | Docker（Redis + RabbitMQ）、**Controller（最后起）** |
| **Person C** | 知识源 + 小车 | TaskConfigurator、Navigator、TargetPlanner、StrategySupervisor、Car001～003 |
| **Person D** | 展示 | SQL Server、Display、浏览器点「开始」 |

硬约束：

- 全组**只有一套** Redis + RabbitMQ（Person A 的 Docker）
- 全组**只有一个** Controller（Person A，且必须最后起）
- 全组**只有一个** Display（Person D）
- 同一辆车不要在两台机器各起一个进程

---

## 2. 前置条件

### 2.1 Tailscale 已互通

三台均安装 Tailscale，通过**邀请链接**加入同一 tailnet（各用各的账号）。

每台执行：

```powershell
tailscale status
tailscale ip -4
```

应能在 `tailscale status` 中看到另外两台在线。

互 ping 测试（把 IP 换成对方的）：

```powershell
ping <对方 Tailscale IP>
```

### 2.2 代码已编译

每台至少执行一次：

```powershell
cd D:\car_homework
.\mvnw.cmd install -DskipTests
```

### 2.3 远程连接参数（推荐，无需 portproxy）

各模块 `main` 支持（`common` 模块 `InfraConnectionConfig`）：

```text
--redis-host <IP_A>   --mq-host <IP_A>
```

| 人员 | 启动时要不要加参数 |
|------|-------------------|
| **Person A** | **不用**（默认 `localhost`，Docker 在本机） |
| **Person C、Person D** | **要**，指向 Person A 的 Tailscale IP |

Person C/D 在 PowerShell 先设变量（`IP_A` 换成 Person A 真实地址）：

```powershell
$IP_A = "100.94.124.1"
$REMOTE = "--redis-host $IP_A --mq-host $IP_A"
```

验证能连 Person A：

```powershell
Test-NetConnection $IP_A -Port 6379
Test-NetConnection $IP_A -Port 5672
```

> 旧方案 **portproxy** 仍可用，见附录 B；优先用 `--redis-host`，更稳。

---

## 3. 启动总顺序

```
① Person A   Docker（Redis + MQ）+ 防火墙
② Person C   带 $REMOTE 起 7 个模块
③ Person D   带 $REMOTE 起 Display
④ Person A   最后起 Controller（无参数）
⑤ Person D   浏览器 localhost:8887 → 点「开始」
```

---

## 4. Person A 的操作（基础设施机）

### 4.1 查 Tailscale IP 并发到群里

```powershell
tailscale ip -4
```

告知 Person C、Person D：**「基础设施 IP 是 xxx，portproxy 填这个」**。

### 4.2 启动 Docker

```powershell
cd D:\car_homework
docker compose up -d
docker ps
```

应看到 `redis`、`rabbitmq` 两个容器在运行。

### 4.3 本机验证端口（PowerShell）

```powershell
Test-NetConnection 127.0.0.1 -Port 6379
Test-NetConnection 127.0.0.1 -Port 5672
```

`TcpTestSucceeded : True` 即正常。

> 若在 cmd 中执行，请用：  
> `powershell -Command "Test-NetConnection 127.0.0.1 -Port 6379"`

### 4.4 防火墙放行（管理员 PowerShell，只需做一次）

```powershell
New-NetFirewallRule -DisplayName "Redis6379" -Direction Inbound -Protocol TCP -LocalPort 6379 -Action Allow
New-NetFirewallRule -DisplayName "RabbitMQ5672" -Direction Inbound -Protocol TCP -LocalPort 5672 -Action Allow
New-NetFirewallRule -DisplayName "RabbitMQ15672" -Direction Inbound -Protocol TCP -LocalPort 15672 -Action Allow
```

### 4.5 等待 Person C、Person D 就绪

在群里确认：

- Person C：工作模块已全部启动
- Person D：Display 已启动

### 4.6 最后启动 Controller（仅 Person A）

```powershell
cd D:\car_homework
Start-Process cmd -ArgumentList '/k','title Controller && .\mvnw.cmd exec:java -pl controller -Dexec.mainClass=com.substation.controller.ControllerMain'
```

> **PowerShell 注意**：不要用 `start "xxx" cmd /k ...`（那是 cmd 语法，PowerShell 会报错）。也可改用 **命令提示符（cmd）** 执行下方「附录 A」中的 cmd 版命令。

成功标志：

```text
[Controller] 控制器已启动，等待任务配置...
```

---

## 5. Person C 的操作（规划 + 小车）

### 5.1 设置远程参数

```powershell
$IP_A = "100.94.124.1"    # 换成 Person A 的 Tailscale IP
$REMOTE = "--redis-host $IP_A --mq-host $IP_A"
Test-NetConnection $IP_A -Port 6379
Test-NetConnection $IP_A -Port 5672
```

确认本机 **未** 运行自己的 Docker Redis/MQ：`docker ps` 中不应有 redis/rabbitmq。

### 5.2 启动模块

```powershell
cd D:\car_homework

Start-Process cmd -ArgumentList '/k',"title TaskConfigurator && .\mvnw.cmd exec:java -pl task-configurator -Dexec.mainClass=com.substation.taskconfigurator.TaskConfiguratorMain `"-Dexec.args=$REMOTE`""
Start-Sleep -Seconds 3

Start-Process cmd -ArgumentList '/k',"title Navigator && .\mvnw.cmd exec:java -pl navigator -Dexec.mainClass=com.substation.navigator.NavigatorMain `"-Dexec.args=$REMOTE`""
Start-Process cmd -ArgumentList '/k',"title TargetPlanner && .\mvnw.cmd exec:java -pl target-planner -Dexec.mainClass=com.substation.targetplanner.TargetPlannerMain `"-Dexec.args=$REMOTE`""
Start-Process cmd -ArgumentList '/k',"title StrategySupervisor && .\mvnw.cmd exec:java -pl strategy-supervisor -Dexec.mainClass=com.substation.strategysupervisor.StrategySupervisorMain `"-Dexec.args=$REMOTE`""

Start-Process cmd -ArgumentList '/k',"title Car001 && .\mvnw.cmd exec:java -pl car -Dexec.mainClass=com.substation.car.CarMain `"-Dexec.args=Car001 $REMOTE`""
Start-Process cmd -ArgumentList '/k',"title Car002 && .\mvnw.cmd exec:java -pl car -Dexec.mainClass=com.substation.car.CarMain `"-Dexec.args=Car002 $REMOTE`""
Start-Process cmd -ArgumentList '/k',"title Car003 && .\mvnw.cmd exec:java -pl car -Dexec.mainClass=com.substation.car.CarMain `"-Dexec.args=Car003 $REMOTE`""
```

若 PowerShell 引号不便，可改用 **cmd**（附录 A，已含 `$REMOTE` 示例）。

共 7 个 cmd 窗口；启动日志中应看到连向 **`100.x.x.x:6379`**，而不是 `localhost`。群里回复：**「工作机已就绪」**。

## 6. Person D 的操作（Display）

### 6.1 设置远程参数

与 Person C 相同：

```powershell
$IP_A = "100.94.124.1"
$REMOTE = "--redis-host $IP_A --mq-host $IP_A"
```

### 6.2 启动 SQL Server

登录功能依赖 SQL Server，须先确保本机 SQL Server 已运行。

### 6.3 启动 Display

```powershell
cd D:\car_homework
Start-Process cmd -ArgumentList '/k',"title Display && .\mvnw.cmd exec:java -pl display -Dexec.mainClass=com.substation.display.DisplayMain `"-Dexec.args=$REMOTE`""
```

群里回复：**「Display 已就绪」**。

### 6.4 打开浏览器（Controller 起来之后）

Person A 起好 Controller 后，Person D 打开：

```text
http://localhost:8887
```

登录 → 进入仿真页 → **只由 Person D 一人**点「开始」。

## 7. 联调检查清单

| 步骤 | 谁检查 | 命令 / 现象 |
|------|--------|-------------|
| Tailscale 互通 | 全员 | `tailscale status` 见另外两台 |
| Redis/MQ 运行 | Person A | `docker ps` |
| 本机 6379 通 | Person A | `Test-NetConnection 127.0.0.1 -Port 6379` |
| 远程能连 Redis/MQ | Person C、D | `Test-NetConnection IP_A -Port 6379` |
| 模块连对地址 | Person C、D | 启动日志为 `IP_A:6379`，非 localhost |
| MQ 管理台 | 任意 | `http://IP_A:15672`（guest/guest） |
| 工作模块 | Person C | 7 个窗口无报错 |
| Display | Person D | `http://localhost:8887` 能打开 |
| Controller 唯一 | Person A | 仅 Person A 一台；日志「等待任务配置」 |
| 仿真运行 | Person D | 点开始后地图刷新、小车移动 |
| 联调结束恢复单机 | Person C、D | 去掉 `-Dexec.args` 中的远程参数，本机 `docker compose up -d` |

## 8. 常见问题

| 现象 | 原因 | 处理 |
|------|------|------|
| `Test-NetConnection` 不是命令 | 在 cmd 而非 PowerShell | 打开 PowerShell，或用 `powershell -Command "..."` |
| `start "xxx" cmd /k` 报错 | 在 PowerShell 里用了 cmd 的 `start` | 用本文 `Start-Process cmd -ArgumentList ...`，或改开 **cmd** 执行附录 A |
| ping 不通 | Tailscale 未连接 / 未进同一 tailnet | 检查托盘 Connected；重开邀请链接 |
| 连不上 `IP_A:6379` | Person A 防火墙或 Docker 未起 | Person A 检查 `docker ps`、防火墙规则 |
| 本机 6379 失败 | 未加 `--redis-host` 或 IP 错 | 检查 `$REMOTE` 与 `Test-NetConnection IP_A` |
| Controller 秒退 | 已有另一个 Controller | 全组只保留 Person A 一个 |
| 车不动 | Controller 起太早或未起 | 确认顺序：Person C、D 就绪后 Person A 再起 |
| 页面无数据 | Display 未起或 WS 断开 | Person D 用 `localhost:8887` |
| 登录失败 | SQL Server 未在 Person D 运行 | Person D 启动 SQL Server |
| 联调后本机单机跑不起来 | 仍带 `$REMOTE` 启动 | 单机启动时不传 `--redis-host`，并起本机 Docker |

## 9. 联调结束后恢复单机（Person C、Person D）

分布式联调时启动命令带了 `--redis-host IP_A`。恢复单机时：

1. 启动命令**去掉** `$REMOTE` / `--redis-host` 参数（默认连 `localhost`）
2. 本机执行：

```powershell
cd D:\car_homework
docker compose up -d
.\start_all.bat
```

若曾配置过 portproxy（附录 B），联调结束也请删除。

---

## 10. 远程看页面的说明

Person D 必须用 **`http://localhost:8887`** 操作仿真。

若 Person A 或 Person C 访问 `http://<Person D 的 Tailscale IP>:8887`，页面能开，但 WebSocket 写死 `localhost:8888`，地图可能不同步。答辩演示建议 Person D 本机操作，或远程桌面到 Person D。

## 11. 附录 A：cmd 版启动命令（Person C 分布式）

将 `100.94.124.1` 换成 Person A 的 Tailscale IP：

```cmd
cd /d D:\car_homework
set IP_A=100.94.124.1
set REMOTE=--redis-host %IP_A% --mq-host %IP_A%

start "TaskConfigurator" cmd /k .\mvnw.cmd exec:java -pl task-configurator -Dexec.mainClass=com.substation.taskconfigurator.TaskConfiguratorMain -Dexec.args="%REMOTE%"
timeout /t 3

start "Navigator" cmd /k .\mvnw.cmd exec:java -pl navigator -Dexec.mainClass=com.substation.navigator.NavigatorMain -Dexec.args="%REMOTE%"
start "TargetPlanner" cmd /k .\mvnw.cmd exec:java -pl target-planner -Dexec.mainClass=com.substation.targetplanner.TargetPlannerMain -Dexec.args="%REMOTE%"
start "StrategySupervisor" cmd /k .\mvnw.cmd exec:java -pl strategy-supervisor -Dexec.mainClass=com.substation.strategysupervisor.StrategySupervisorMain -Dexec.args="%REMOTE%"

start "Car001" cmd /k .\mvnw.cmd exec:java -pl car -Dexec.mainClass=com.substation.car.CarMain -Dexec.args="Car001 %REMOTE%"
start "Car002" cmd /k .\mvnw.cmd exec:java -pl car -Dexec.mainClass=com.substation.car.CarMain -Dexec.args="Car002 %REMOTE%"
start "Car003" cmd /k .\mvnw.cmd exec:java -pl car -Dexec.mainClass=com.substation.car.CarMain -Dexec.args="Car003 %REMOTE%"
```

Person D Display：

```cmd
start "Display" cmd /k .\mvnw.cmd exec:java -pl display -Dexec.mainClass=com.substation.display.DisplayMain -Dexec.args="%REMOTE%"
```

Person A Controller（无远程参数）：

```cmd
start "Controller" cmd /k .\mvnw.cmd exec:java -pl controller -Dexec.mainClass=com.substation.controller.ControllerMain
```

## 12. 附录 B：portproxy 方案（旧，可选）

若暂时无法 `git pull` 到带 `--redis-host` 的版本，仍可用 portproxy。Person C、D 管理员 PowerShell：

```powershell
netsh interface portproxy add v4tov4 listenaddress=127.0.0.1 listenport=6379 connectaddress=IP_A connectport=6379
netsh interface portproxy add v4tov4 listenaddress=127.0.0.1 listenport=5672 connectaddress=IP_A connectport=5672
```

联调结束后删除：

```powershell
netsh interface portproxy delete v4tov4 listenaddress=127.0.0.1 listenport=6379
netsh interface portproxy delete v4tov4 listenaddress=127.0.0.1 listenport=5672
```

## 一、功能概述

仿真系统在 Controller 判定探索完成（100%）时，前端弹出保存窗口，用户可选择保存仿真记录。管理员和统计分析员可在统计分析页面查看已保存的仿真记录详情。

---

## 二、数据流

```
Controller 判定完成(100%)
  → WebSocket REFRESH_ALL 广播
    → app.js updateGlobalInfo() 检测 explorationRate >= 100
      → showSavePopup() 弹窗 "是否保存？"
        ├─ 保存 → 数据写入 localStorage(sim_records)
        └─ 不保存 → 丢弃
      
analysis.html(统计分析页)
  → analysis.js loadList()
    → 读取 localStorage(sim_records)
      → 渲染记录卡片
        → 点击记录 → showDetail() 全屏展示图表
```

---

## 五、统计分析界面

### 5.1 列表页

- 卡片网格布局，横向铺满
- 每个卡片显示：记录编号、日期时间、探索率、步数、耗时、车辆数
- 右上角 ✕ 按钮可删除记录
- 粒子背景动画

### 5.2 详情页（全屏）

点击卡片 → 全屏展示：

| 区域 | 内容 |
|------|------|
| 顶部 | 4 个指标卡片（探索率/总步数/有效步数/耗时） |
| 第二行 | 柱状图：各车总步数 + 各车有效步数 |
| 第三行 | 柱状图：关键指标对比 + 进度条：探索进度 |
| 右上角 | ✕ 关闭按钮，点击返回列表 |

## 六、完成弹窗逻辑

- Controller 通过 WebSocket 广播 explorationRate >= 100 时触发
- 弹窗显示探索率、耗时
- 2 个按钮：💾 保存记录 / 🗑 不保存
- 保存后记录追加到 localStorage，最多保留 50 条

## 七、技术要点

| 项目 | 方案 |
|------|------|
| 图表渲染 | 纯 Canvas（无 CDN 依赖） |
| 数据存储 | localStorage |
| 粒子背景 | Canvas 固定定位 z-index:0 |
| token 续期 | 仿真页每 10 分钟自动调用 /api/auth/me |
| 返回导航 | 所有页面返回均跳转 dashboard.html |

# 四人四机分布式启动流程（Tailscale）

> **用途**：四人四机联调、答辩演示时的操作清单。  
> **网络**：Tailscale 同一 tailnet，全程使用 `100.x.x.x` 地址。  
> **更新**：2026-06-23  
> **配置**：`deploy/infra.local.json` + `scripts/*.ps1`（无需每次手敲 `$IP_A` / `$REMOTE`）

---

## 1. 分工一览

| 人员 | 机器职责 | 关键进程 |
|------|----------|----------|
| **Person A** | 基础设施 + 调度 | Docker（Redis + RabbitMQ）、**Controller（最后起）** |
| **Person B** | 小车 | Car001、Car002、Car003 |
| **Person C** | 知识源（规划） | TaskConfigurator、Navigator、TargetPlanner、StrategySupervisor |
| **Person D** | 展示 | SQL Server、Display、浏览器点「开始」 |

硬约束：

- 全组**只有一套** Redis + RabbitMQ（Person A 的 Docker）
- 全组**只有一个** Controller（Person A，且必须最后起）
- 全组**只有一个** Display（Person D）
- 同一辆车不要在两台机器各起一个进程

---

## 2. 前置条件

### 2.1 Tailscale 已互通

四台均安装 Tailscale，通过**邀请链接**加入同一 tailnet（各用各的账号）。

每台执行：

```powershell
tailscale status
tailscale ip -4
```

应能在 `tailscale status` 中看到另外三台在线。

互 ping 测试（把 IP 换成对方的）：

```powershell
ping <对方 Tailscale IP>
```

### 2.2 代码已编译

每台至少执行一次：

```powershell
cd D:\car_homework
.\mvnw.cmd install -DskipTests
```

### 2.3 本地部署配置（推荐）

各模块启动时自动读取 **`deploy/infra.local.json`**（`InfraConnectionConfig.resolve`）。  
**首次**在本机项目根执行一次配置脚本即可，之后用角色启动脚本分窗起进程。

#### Person A（基础设施，localhost）

```powershell
cd D:\car_homework
.\scripts\setup-config.ps1 -Role infra
```

#### Person B / C / D（远程连 Person A）

将 `100.94.124.1` 换成 Person A 的 Tailscale IP：

```powershell
cd D:\car_homework

# Person C
.\scripts\setup-config.ps1 -Remote -InfraHost 100.94.124.1 -Role planner

# Person B
.\scripts\setup-config.ps1 -Remote -InfraHost 100.94.124.1 -Role car

# Person D（DisplayHost 填本机 Tailscale IP，供他人浏览器访问）
.\scripts\setup-config.ps1 -Remote -InfraHost 100.94.124.1 -Role display -DisplayHost 100.94.124.4
```

配置文件会生成到 `deploy/infra.local.json`（已加入 `.gitignore`，不会提交个人 IP）。

| 字段 | 说明 |
|------|------|
| `redisHost` / `mqHost` | Person A 的 IP；A 机为 `localhost` |
| `role` | `infra` / `planner` / `car` / `display`（脚本提示用） |
| `cars` | Person B 启动的小车 ID 列表，默认 `Car001`～`Car003` |

验证能连 Person A：

```powershell
Test-NetConnection 100.94.124.1 -Port 6379
Test-NetConnection 100.94.124.1 -Port 5672
```

> 仍支持命令行 `--redis-host` / `--mq-host`（覆盖配置文件）。旧方案 **portproxy** 见附录 B。

### 2.4 一键启动脚本

| 人员 | 脚本 | 作用 |
|------|------|------|
| Person A | `.\scripts\start-infra.ps1` | Docker + 提示最后起 Controller |
| Person A | `.\scripts\start-controller.ps1` | **最后**起 Controller |
| Person C | `.\scripts\start-planner.ps1` | 4 个规划模块 |
| Person B | `.\scripts\start-cars.ps1` | `cars` 列表中的小车 |
| Person D | `.\scripts\start-display.ps1` | Display |
| 任意 | `.\scripts\start-navigator.ps1 -Count 2` | 额外 Navigator 扩算力 |

## 3. 启动总顺序

```
① Person A   setup-config → start-infra.ps1（Docker + 防火墙）
② Person C   setup-config → start-planner.ps1
③ Person B   setup-config → start-cars.ps1（可与 ② 并行）
④ Person D   setup-config → start-display.ps1
⑤ Person A   start-controller.ps1（最后）
⑥ Person D   浏览器 localhost:8887 → 点「开始」
```

## 4. Person A 的操作（基础设施机）

### 4.1 查 Tailscale IP 并发到群里

```powershell
tailscale ip -4
```

告知 Person B、C、D：**「基础设施 IP 是 xxx」**。

### 4.2 配置并启动 Docker

```powershell
cd D:\car_homework
.\scripts\setup-config.ps1 -Role infra
.\scripts\start-infra.ps1
```

或手动：

```powershell
docker compose up -d
docker ps
```
```

应看到 `redis`、`rabbitmq` 两个容器在运行。

### 4.3 本机验证端口（PowerShell）

```powershell
Test-NetConnection 127.0.0.1 -Port 6379
Test-NetConnection 127.0.0.1 -Port 5672
```

`TcpTestSucceeded : True` 即正常。

> 若在 cmd 中执行，请用：  
> `powershell -Command "Test-NetConnection 127.0.0.1 -Port 6379"`

### 4.4 防火墙放行（管理员 PowerShell，只需做一次）

```powershell
New-NetFirewallRule -DisplayName "Redis6379" -Direction Inbound -Protocol TCP -LocalPort 6379 -Action Allow
New-NetFirewallRule -DisplayName "RabbitMQ5672" -Direction Inbound -Protocol TCP -LocalPort 5672 -Action Allow
New-NetFirewallRule -DisplayName "RabbitMQ15672" -Direction Inbound -Protocol TCP -LocalPort 15672 -Action Allow
```

### 4.5 等待 Person B、C、D 就绪

在群里确认：

- Person C：4 个规划模块已全部启动
- Person B：3 个小车进程已全部启动
- Person D：Display 已启动

### 4.6 最后启动 Controller（仅 Person A）

```powershell
cd D:\car_homework
.\scripts\start-controller.ps1
```

成功标志：

```text
[Controller] 控制器已启动，等待任务配置...
```

## 5. Person C 的操作（规划模块）

### 5.1 配置（首次）

```powershell
cd D:\car_homework
.\scripts\setup-config.ps1 -Remote -InfraHost 100.94.124.1 -Role planner
Test-NetConnection 100.94.124.1 -Port 6379

确认本机 **未** 运行自己的 Docker Redis/MQ：`docker ps` 中不应有 redis/rabbitmq。

### 5.2 启动模块

```powershell

```

共 **4** 个 cmd 窗口；启动日志中应看到连向 Person A 的 **`100.x.x.x:6379`**。群里回复：**「规划模块已就绪」**。

## 6. Person B 的操作（小车）

### 6.1 配置（首次）

```powershell
cd D:\car_homework
.\scripts\setup-config.ps1 -Remote -InfraHost 100.94.124.1 -Role car
```

可在 `deploy/infra.local.json` 的 `cars` 数组中增减车辆（如加 `Car004`）。

## 7. Person D 的操作（Display）

### 7.1 配置（首次）

```powershell

.\scripts\setup-config.ps1 -Remote -InfraHost 100.94.124.1 -Role display -DisplayHost <本机TailscaleIP>

### 7.2 启动 SQL Server

登录功能依赖 SQL Server，须先确保本机 SQL Server 已运行。

### 7.3 启动 Display

```powershell

```

群里回复：**「Display 已就绪」**。

Person A 起好 Controller 后，Person D 打开：

```text
http://localhost:8887
```

登录 → 进入仿真页 → **只由 Person D 一人**点「开始」。

## 8. 联调检查清单

| 步骤 | 谁检查 | 命令 / 现象 |
|------|--------|-------------|
| Tailscale 互通 | 全员 | `tailscale status` 见另外三台 |
| Redis/MQ 运行 | Person A | `docker ps` |
| 本机 6379 通 | Person A | `Test-NetConnection 127.0.0.1 -Port 6379` |
| 远程能连 Redis/MQ | Person B、C、D | `Test-NetConnection IP_A -Port 6379` |
| 模块连对地址 | Person B、C、D | 启动日志为 `IP_A:6379`，非 localhost |
| MQ 管理台 | 任意 | `http://IP_A:15672`（guest/guest） |
| 规划模块 | Person C | 4 个窗口无报错 |
| 小车 | Person B | 3 个窗口无报错 |
| Display | Person D | `http://localhost:8887` 能打开 |
| Controller 唯一 | Person A | 仅 Person A 一台；日志「等待任务配置」 |
| 仿真运行 | Person D | 点开始后地图刷新、小车移动 |
| 联调结束恢复单机 | Person B、C、D | 删除或改回 `deploy/infra.local.json` 为 localhost，本机 `docker compose up -d` 后 `.\start_all.bat` |

## 9. 常见问题

| 现象 | 原因 | 处理 |
|------|------|------|
| `Test-NetConnection` 不是命令 | 在 cmd 而非 PowerShell | 打开 PowerShell，或用 `powershell -Command "..."` |
| `start "xxx" cmd /k` 报错 | 在 PowerShell 里用了 cmd 的 `start` | 用本文 `Start-Process cmd -ArgumentList ...`，或改开 **cmd** 执行附录 A |
| ping 不通 | Tailscale 未连接 / 未进同一 tailnet | 检查托盘 Connected；重开邀请链接 |
| 连不上 `IP_A:6379` | Person A 防火墙或 Docker 未起 | Person A 检查 `docker ps`、防火墙规则 |
| 本机 6379 失败 | 配置文件 IP 错或未创建 | 检查 `deploy/infra.local.json` 与 `Test-NetConnection IP_A` |
| Controller 秒退 | 已有另一个 Controller | 全组只保留 Person A 一个 |
| 车不动 | Controller 起太早或未起 | 确认 B、C、D 就绪后 Person A 再起 Controller |
| 部分车不动 | Person B 未起对应 Car 进程 | 检查 Car004 等是否在 B 机启动 |
| 页面无数据 | Display 未起或 WS 断开 | Person D 用 `localhost:8887` |
| 登录失败 | SQL Server 未在 Person D 运行 | Person D 启动 SQL Server |
| 联调后本机单机跑不起来 | `infra.local.json` 仍为远程 IP | 运行 `.\scripts\setup-config.ps1 -Role infra` 或改回 localhost |

## 10. 联调结束后恢复单机（Person B、C、D）

分布式联调时使用 `deploy/infra.local.json`。恢复单机时：

1. 运行 `.\scripts\setup-config.ps1 -Role infra`，或手动把 `redisHost` / `mqHost` 改回 `localhost`
2. 本机执行：

```powershell
cd D:\car_homework
docker compose up -d
.\start_all.bat
```

若曾配置过 portproxy（附录 B），联调结束也请删除。

## 12. 附录 A：手动 cmd 启动（备用）

set REMOTE=--redis-host %IP_A% --mq-host %IP_A%

```cmd
start "TaskConfigurator" cmd /k .\mvnw.cmd exec:java -pl task-configurator -Dexec.mainClass=com.substation.taskconfigurator.TaskConfiguratorMain -Dexec.args="%REMOTE%"

start "Navigator" cmd /k .\mvnw.cmd exec:java -pl navigator -Dexec.mainClass=com.substation.navigator.NavigatorMain -Dexec.args="%REMOTE%"

start "StrategySupervisor" cmd /k .\mvnw.cmd exec:java -pl strategy-supervisor -Dexec.mainClass=com.substation.strategysupervisor.StrategySupervisorMain -Dexec.args="%REMOTE%"
```

### Person B — 小车

```cmd
start "Car001" cmd /k .\mvnw.cmd exec:java -pl car -Dexec.mainClass=com.substation.car.CarMain -Dexec.args="Car001 %REMOTE%"

## 13. 附录 B：portproxy 方案（旧，可选）

若暂时无法 `git pull` 到带 `--redis-host` 的版本，仍可用 portproxy。Person B、C、D 管理员 PowerShell：

```powershell
netsh interface portproxy add v4tov4 listenaddress=127.0.0.1 listenport=6379 connectaddress=IP_A connectport=6379
netsh interface portproxy add v4tov4 listenaddress=127.0.0.1 listenport=5672 connectaddress=IP_A connectport=5672
```

联调结束后删除：

```powershell
netsh interface portproxy delete v4tov4 listenaddress=127.0.0.1 listenport=6379
netsh interface portproxy delete v4tov4 listenaddress=127.0.0.1 listenport=5672
```

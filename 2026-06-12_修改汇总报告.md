## 一、Common 模块

### 1.1 pushRoute 逆序推入
**来源**：Person C 方案，Person A 实施  
**文件**：`common/.../BlackboardClient.java`

原 LPUSH 正序推入 `[A,B,C]` 得 `[C,B,A]`，Car 从尾部读正确，Display 从头部读反转（路线显示颠倒/斜穿障碍物的根因）。改为逆序 `i--` 推入，Redis 列表变为 `[A,B,C]`，Car 与 Display 均从头部读，顺序一致。

### 1.2 peek/pop 方向配合
**来源**：Person C 方案，Person A 实施  
**文件**：`common/.../BlackboardClient.java`

`peekNextRouteStep`：`LINDEX -1` → `LINDEX 0`（从尾部 peek 改为从头部 peek）  
`popNextRouteStep`：`RPOP` → `LPOP`（从尾部消费改为从头部消费）

### 1.3 setElapsedSeconds
**来源**：Person C 方案，Person A 实施  
**文件**：`common/.../BlackboardClient.java`

新增 `setElapsedSeconds(long)` 方法，单字段 HSET 写入 `elapsedSeconds`，不覆盖 TaskConfig 其他字段。

### 1.4 位置预约锁
**来源**：Person B 开发，Person A 加入 common  
**文件**：`common/.../BlackboardClient.java`

新增 `tryReservePosition(x, y, carId)`：`SET NX EX 3` 预约目标格，防止两车同时移动到同一格。  
新增 `releaseReservePosition(x, y, carId)`：校验 carId 匹配后 DEL 释放。

## 二、Controller 模块

### 2.1 重置流程修复
**来源**：Person A 自修  
**文件**：`controller/.../TickScheduler.java`、`StatusDispatcher.java`、`CommandHandler.java`

| 问题 | 修复 |
|------|------|
| `forwardReset` 调 `scheduler.stop()` 关闭线程池，后续 `start()` 失败 | `stop()` 改为只取消 future 不关线程池，新增 `shutdown()` 仅在 JVM 退出时用 |
| 暂停后重置，`paused=true` 残留，tickLoop 跳过所有 tick | `resetPaused()` 在 `forwardReset` 中清零 |
| 暂停/重置期间 TARGET_ASSIGNED/ROUTE_PLANNED 照常处理，污染新状态 | `CommandHandler` 加 `isPaused()` 和 `isActive()` 双重守卫 |
| `ROUTE_PLANNED` 无去重保护 | 新增 `pendingPlanRequests` 集合，`checkAndPlanRoute` 同车只发一次 |

### 2.2 pendingPlanRequests 去重 + 配合 onRoutePlanned
**来源**：Person C 方案，Person A 实施  
**文件**：`controller/.../StatusDispatcher.java`

`checkAndPlanRoute`：`pendingPlanRequests.add(carId)` 失败则跳过  
`onRoutePlanned`：`pendingPlanRequests.remove(carId)`  
`forwardReset`：`pendingPlanRequests.clear()`

### 2.3 completeTask 改用 setElapsedSeconds
**来源**：Person C 方案，Person A 实施  
**文件**：`controller/.../StatusDispatcher.java`

`initTaskConfig(Map.of(...))` → `setElapsedSeconds(elapsed)`。

### 2.4 TaskConfigurator 重置改 FLUSHDB
**来源**：Person A 自修  
**文件**：`task-configurator/.../TaskConfiguratorMain.java`

`selectiveClear`（KEYS + DEL）改为 `FLUSHDB`，彻底清空 Redis 防止 Car 进程竞态重建 key。`handleReset` 不再发 TASK_READY。

# 分布式手动启动命令（全组 cmd 版）

> **适用场景**：`start-*.ps1` 脚本报 `ClassNotFoundException`、或 Maven 在根项目 `@ car-homework` 上执行 `exec:java` 时，用本文档**手敲命令**联调。  
> **网络**：同一 WiFi 见 `同一WiFi分布式联调指南.md`；Tailscale 见 `分布式联调使用指南.md`。  
> **配置**：各模块自动读取 `deploy/infra.local.json`（B/C/D 的 `redisHost` / `mqHost` 填 Person A 的 IP）。

---

## 核心规则

每条模块启动都是 **两步**，且必须在 **cmd** 里执行（避免 PowerShell 吞掉 `-D` 参数）：

```cmd
.\mvnw.cmd -pl <模块名> -am compile -q
.\mvnw.cmd -pl <模块名> exec:java -Dexec.mainClass=<Main类全名> [ -Dexec.args=... ]
```

**成功标志**：日志里出现 `--- exec:java @ <模块名> ---`，例如 `@ display`、`@ car`。  
**失败标志**：出现 `@ car-homework` 且 `ClassNotFoundException` → 说明没加 `-pl` 或两步写反了。

将下文所有 `D:\car_homework` 替换为各自本机项目路径。

---

## 0. 四人先做（任意角色，做一次即可）

```cmd
cd /d D:\car_homework
.\mvnw.cmd install -DskipTests
```

确认 `deploy\infra.local.json` 已按角色配置（见 `分布式联调使用指南.md` §8）。

## 1. 全组启动顺序

```
Person A   docker compose up -d
Person C   §4 四个规划模块（4 个 cmd 窗口）
Person B   §3 小车（每辆 1 个 cmd 窗口）     ← 可与 C 并行
Person D   §5 Display（1 个 cmd 窗口）
           → 群里确认 B/C/D「已就绪」
Person A   §2 Controller（1 个 cmd，必须最后）
Person D   浏览器 http://localhost:8887 点「开始」
其他人     http://<D的IP>:8887 观看
```

**不要用 `start_all.bat` 做四人分布式。**

## 2. Person A — Docker + Controller

### 2.1 起 Docker（一个 cmd，执行完可关）

```cmd
cd /d D:\car_homework
docker compose up -d
docker ps
```

应看到 redis、rabbitmq 容器在运行。

### 2.2 防火墙（管理员 PowerShell，本机做一次）

```powershell
New-NetFirewallRule -DisplayName "Redis6379" -Direction Inbound -Protocol TCP -LocalPort 6379 -Action Allow
New-NetFirewallRule -DisplayName "RabbitMQ5672" -Direction Inbound -Protocol TCP -LocalPort 5672 -Action Allow
```

### 2.3 Controller（等 B/C/D 就绪后再开，单独 cmd，保持运行）

```cmd
cd /d D:\car_homework
.\mvnw.cmd -pl controller -am compile -q
.\mvnw.cmd -pl controller exec:java -Dexec.mainClass=com.substation.controller.ControllerMain
```

成功日志：`[Controller] 控制器已启动，等待任务配置...`

`infra.local.json` 中 A 的 `redisHost` / `mqHost` 应为 `localhost`。

## 7. 30 秒自检

| 检查项 | 命令 / 现象 |
|--------|-------------|
| B/C/D 能连 A 的 Redis | `Test-NetConnection <IP_A> -Port 6379` → `TcpTestSucceeded : True` |
| A 的 Docker | `docker ps` 有 redis、rabbitmq |
| 模块起对了 | 日志是 `@ display` / `@ car` 等，不是 `@ car-homework` |
| B 的车 | 日志为 `IP_A:6379` |
| D 的页面 | `http://localhost:8887` 能打开 |
| 仿真跑起来 | D 点开始后探索率上涨、小车移动 |

## 8. 常见问题

| 现象 | 处理 |
|------|------|
| `ClassNotFoundException: *Main` | 未加 `-pl`；按本文「两步」重开 |
| 日志是 `@ car-homework` | 同上；勿用旧版 `exec:java -pl xxx -am` 单条命令 |
| `install` 未做过 | 执行 §0 的 `mvnw install -DskipTests` |
| 车不动 | A 是否**最后**起了 Controller；B 车是否已起 |
| 登录失败 | D 本机 SQL Server 未启动 |
| 连不上 Redis | 检查 `infra.local.json`、A 防火墙、A 的 Docker |
| 脚本修好后 | 可改回 `scripts\start-*.ps1`（需含 `compile` 再 `exec:java` 两段的版本） |

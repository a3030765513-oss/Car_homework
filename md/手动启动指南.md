# 手动启动指南

> 单机联调与分布式多机均适用。项目根目录：`D:\car_homework`（下文命令均在此目录执行）。

---

## 一、启动前配置（一次性 / 换机器时做）

### 1.1 基础设施

| 组件 | 用途 | 默认地址 |
|------|------|----------|
| **Redis** | 黑板（地图、车辆状态） | `localhost:6379` |
| **RabbitMQ** | 模块间 MQ | `localhost:5672`（管理台 `http://localhost:15672`，`guest`/`guest`） |
| **SQL Server** | 登录、统计分析、场次存档（仅 Display 需要） | `localhost:1433`，库名 `CarHomework` |

启动 Redis + RabbitMQ（Docker）：

```powershell
cd D:\car_homework
docker compose up -d
```

确认端口就绪：`6379`、`5672`、`15672` 可访问。

### 1.2 SQL Server（Display 必开）

Display 启动时会连 JDBC 并自动建表、写入默认管理员。

| 项 | 默认值（见 `DatabaseManager.java`） |
|----|--------------------------------------|
| URL | `jdbc:sqlserver://localhost:1433;databaseName=CarHomework;encrypt=false;trustServerCertificate=true` |
| 用户 | `sa` |
| 密码 | `Hospital#2024` |

浏览器默认管理员：`admin` / `admin123`（Display 日志会提示）。

### 1.3 编译

首次或改代码后：

```powershell
cd D:\car_homework
.\mvnw.cmd install -DskipTests
```

仅编某一模块（示例 controller）：

```powershell
.\mvnw.cmd install -pl controller -am -DskipTests
```

**ADD_CAR 动态加车** 还需 car 的 fat jar：

```powershell
.\mvnw.cmd package -pl car -am -DskipTests
```

产物：`car\target\car-1.0-SNAPSHOT.jar`（小车进程在跑时勿 `clean`，可用 `install -rf :car`）。

### 1.4 部署配置 `deploy/infra.local.json`

各模块 `main` 通过 `InfraConnectionConfig.resolve(args)` 读此文件（**不提交 Git**，每人本地一份）。

**单机**：复制模板即可：

```powershell
copy deploy\infra.example.json deploy\infra.local.json
```

`infra.example.json` 示例：

```json
{
  "redisHost": "localhost",
  "redisPort": 6379,
  "mqHost": "localhost",
  "mqPort": 5672,
  "role": "infra",
  "displayHost": "localhost",
  "displayHttpPort": 8887,
  "displayWsPort": 8888,
  "cars": ["Car001", "Car002", "Car003"]
}
```

**分布式**（多机指向 Person A 的 Redis/MQ）：

```powershell
.\scripts\setup-config.ps1 -Role planner    # C 机：填 A 的 Tailscale IP
.\scripts\setup-config.ps1 -Role car        # B 机
.\scripts\setup-config.ps1 -Role display    # D 机
.\scripts\setup-config.ps1 -Role infra      # A 机本机
```

或手动复制 `deploy\infra.remote.example.json` → `deploy\infra.local.json`，改 `redisHost`、`mqHost` 为 **A 机 IP**。

**命令行覆盖**（优先级高于 json）：

```text
--redis-host <IP>  --redis-port 6379  --mq-host <IP>  --mq-port 5672
```

示例（远程连 A）：

```powershell
.\mvnw.cmd exec:java -pl car -am "-Dexec.mainClass=com.substation.car.CarMain" "-Dexec.args=Car001 --redis-host 100.x.x.x"
```

### 1.5 控制台编码（Windows 中文日志）

```powershell
chcp 65001
```

或使用项目自带 `start_all.bat` / `run_module.bat`（已设 UTF-8）。

## 二、推荐启动顺序

```
1. docker compose up -d     # Redis + RabbitMQ
2. SQL Server               # 仅要开 Display / 登录时
3. TaskConfigurator         # 必须仅 1 个消费者
4. Navigator
5. TargetPlanner
6. StrategySupervisor
7. Car001 … Car00N          # 每台车 1 个进程
8. Display
9. Controller               # 建议最后；全局仅 1 个实例
```

**铁律**：

- `TaskConfigCmd`、`ControllerCmd` 等队列 **每类业务进程只起 1 个消费者**（多开会抢消息）。
- Controller 通过 Redis `controller:instance` **单实例锁**，重复启动会退出。

## 三、各进程手动启动命令

以下均在 **项目根目录** 执行。每条命令会 **占用一个终端窗口**（`cmd /k` 或 IDE Run）。

通用模板：

```powershell
.\mvnw.cmd exec:java -pl <模块名> -am "-Dexec.mainClass=<入口类全名>" ["-Dexec.args=..."]
```

也可用 `run_module.bat`（会先 `install` 再 `exec:java`）：

```powershell
.\run_module.bat exec:java -pl controller -am "-Dexec.mainClass=com.substation.controller.ControllerMain"
```

---

### 3.1 TaskConfigurator（任务配置）

| 项 | 值 |
|----|-----|
| 模块 | `task-configurator` |
| 入口 | `com.substation.taskconfigurator.TaskConfiguratorMain` |
| 订阅队列 | `TaskConfigCmd`（`FORWARD_CONFIG` / `FORWARD_RESET`） |
| 发出 | `TASK_READY` → `ControllerCmd` |

```powershell

### 3.5 Car（小车，可多实例）

| 项 | 值 |
|----|-----|
| 模块 | `car` |
| 入口 | `com.substation.car.CarMain` |
| 参数 | `Car001`（车号，可 `Car002`…） |
| 订阅 | `Car_{carId}`（`TICK_MOVE`） |

.\mvnw.cmd exec:java -pl car -am "-Dexec.mainClass=com.substation.car.CarMain" "-Dexec.args=Car002"

.\mvnw.cmd exec:java -pl car -am "-Dexec.mainClass=com.substation.car.CarMain" "-Dexec.args=Car003"
```

**页面动态加车**（Display 拉起，无需手敲）：

```text
java -jar car\target\car-1.0-SNAPSHOT.jar Car004 --dynamic
```

动态车在 **运行 Display 的那台机器** 上启动，依赖该机的 `infra.local.json` 连 Redis/MQ。

## 四、浏览器侧「开始」配置（仿真参数）

在 `index.html` 侧栏可配（经 WebSocket `SET_CONFIG` → Controller → TaskConfigurator）：

| 参数 | 说明 | 典型默认 |
|------|------|----------|
| 地图宽/高 | 格子数 | 30 × 30 |
| 小车数量 | TaskConfigurator 注册 `Car001…CarN` | 3～5 |
| 障碍比例 | 随机障碍密度 | 0.15（15%） |
| 算法 | `BFS` / `ASTAR` | BFS |
| 节拍间隔 | tick 毫秒 | 500（可调 100～2000） |

**流程**：

```
点「开始」→ SET_CONFIG → FORWARD_CONFIG → TaskConfigurator 写 Redis
         → TASK_READY → Controller 启动 TickScheduler → 车辆开始调度
```

**重置**：`RESET` → 清空仿真黑板（**不清** `auth:*` 登录会话）→ 需再点「开始」重新初始化。

## 六、启动后检查清单

| 检查项 | 方法 |
|--------|------|
| Redis / MQ | `docker ps`；或访问 `http://localhost:15672` |
| 队列消费者 = 1 | RabbitMQ 管理台 → Queues → `TaskConfigCmd`、`ControllerCmd` 等 **Consumers = 1** |
| 点开始后 TC 响应 | TaskConfigurator 窗口：`初始化完成`、`已发送 TASK_READY` |
| Controller 调度 | Controller 窗口：`收到 TASK_READY，启动调度` |
| 黑板有车辆 | Redis：`Car001:Status`、`TaskConfig` 等键存在 |
| 页面能刷新 | Display 无 SQL 连接错误；浏览器已登录 |

**常见问题**：

- 一直等 `TASK_READY`：多台 TaskConfigurator 或 TC 未启动。  
- 车不动：Controller 未启动或未 `TASK_READY`；Car 进程未起。  
- 分布式 Redis 超时：确认 `infra.local.json` 中 IP、防火墙、Tailscale；A 机 Docker 端口已暴露。

## 七、进程与队列对照表

| 进程 | 主要订阅 | 主要发布 |
|------|----------|----------|
| TaskConfigurator | `TaskConfigCmd` | `TASK_READY` |
| Navigator | `NavigatorCmd` | `ROUTE_PLANNED` |
| TargetPlanner | `TargetPlannerCmd` | `TARGET_ASSIGNED` |
| StrategySupervisor | `StrategySupervisorCmd` | `ROUTE_OPTIMIZED` |
| Car_{id} | `Car_{id}` | `MOVED` / `ROUTE_DONE` / `BLOCKED` |
| Display | `UpdateView` fanout | `ControllerCmd`（转发浏览器命令） |
| Controller | `ControllerCmd` | 各业务队列 + `REFRESH_ALL` fanout |

## 八、分布式四人分工速查

| 人员 | 机器角色 | 启动内容 |
|------|----------|----------|
| A | `infra` | Docker + Controller（最后）+ 可选本机 TC |
| B | `car` | Car001…（`cars` 数组） |
| C | `planner` | TaskConfigurator + Navigator + TargetPlanner + StrategySupervisor |
| D | `display` | Display + SQL Server |

B/C/D 的 `deploy/infra.local.json` 中 `redisHost`、`mqHost` 指向 **A 机 IP**；浏览器访问 `http://<D机 displayHost>:8887`。

# 变电站巡检仿真系统 — 接口文档

> 版本：与当前 `main` 代码一致  
> Display 默认端口：**HTTP 8887**，**WebSocket 8888**  
> 中间件：Redis `6379`，RabbitMQ `5672`（管理台 `15672`）

---

## 1. 总览

系统对外接口分三层：

| 层级 | 协议 | 用途 |
|------|------|------|
| **HTTP REST** | `http://<displayHost>:8887` | 登录、统计、回放列表、管理后台 |
| **WebSocket** | `ws://<displayHost>:8888` | 仿真控制、实时状态推送、当场回放 |
| **MQ + Redis** | RabbitMQ + Redis | 各 Java 模块间命令与黑板状态（无 HTTP） |

### 1.1 认证方式（HTTP / 部分 WebSocket 场景）

除白名单接口外，HTTP 请求需携带：

```http
Authorization: Bearer <token>
```

- `token` 由 `POST /api/auth/login` 返回
- Session 存于 Redis `auth:session:*`（仿真 RESET **不会**清除登录态）

### 1.2 HTTP 白名单（无需 token）

- 静态资源：`/css/`、`/js/`、`/unity/`
- 页面：`/login.html`、`/index.html`、`/dashboard.html`、`/analysis.html`
- `POST /api/auth/login`、`POST /api/auth/register`

## 2. HTTP — 认证 `/api/auth/*`

实现：`common/.../auth/AuthApiHandler.java`

### 2.1 注册

`POST /api/auth/register`

**请求体**

```json
{
  "username": "zhangsan",
  "password": "至少6位",
  "role": "simulator",
  "displayName": "张三"
}
```

- `role` 可选：`simulator` | `analyst`（需管理员审核后生效）

**响应**

```json
{ "success": true, "message": "注册申请已提交，请等待管理员审核" }
```

### 2.2 登录

`POST /api/auth/login`

**请求体**

```json
{ "username": "admin", "password": "admin123" }
```

**成功响应**

```json
{
  "success": true,
  "token": "<uuid>",
  "username": "admin",
  "role": "admin",
  "displayName": "管理员"
}
```

### 2.3 登出

`POST /api/auth/logout`  
Header：`Authorization: Bearer <token>`

### 2.4 当前用户

`GET /api/auth/me`

**成功响应**

```json
{
  "success": true,
  "username": "admin",
  "role": "admin",
  "displayName": "管理员"
}
```

### 2.5 修改密码

`POST /api/auth/change-password`

```json
{
  "oldPassword": "旧密码",
  "newPassword": "新密码至少6位"
}
```

成功后 Session 失效，需重新登录。

## 3. HTTP — 统计分析 `/api/analysis/*`

实现：`common/.../analysis/AnalysisApiHandler.java`  
**均需登录。**

### 3.1 场次统计列表

`GET /api/analysis/records?page=1&size=50`

**响应**

```json
{
  "success": true,
  "records": [ /* simulation_run_stats 关联数据 */ ]
}
```

### 3.2 保存本场统计

`POST /api/analysis/records`

仿真结束后弹窗保存时调用。`saved_by` 由服务端从 token 解析。

**请求体示例**

```json
{
  "explorationRate": 100,
  "tick": 420,
  "duration": 210,
  "totalSteps": 800,
  "totalEffectiveSteps": 520,
  "efficiencyPercent": 65,
  "wastedSteps": 280,
  "carCount": 3,
  "algorithm": "BFS",
  "obstacleRatio": 0.15,
  "mapWidth": 30,
  "mapHeight": 30,
  "balanceScore": 0.92,
  "cars": [ /* 各车步数等 */ ],
  "timestamp": 1710000000000,
  "date": "2026/6/25 10:00:00",
  "runId": 0
}
```

- `runId > 0`：为已有 `simulation_runs` 场次补写统计
- `runId` 省略或 0：由 `SimulationRecordService` 归档并写统计

**成功响应**

```json
{
  "success": true,
  "runId": 29,
  "record": { /* 已存记录 */ }
}
```

### 3.3 放弃保存

`POST /api/analysis/discard`

### 3.4 删除统计记录

`DELETE /api/analysis/records/{runId}`

### 3.5 其他（占位 / 简化实现）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/analysis/summary` | 汇总（当前返回空结构） |
| GET | `/api/analysis/car/{carId}` | 单车统计（占位） |
| GET | `/api/analysis/leaderboard` | 排行榜（占位） |
| POST | `/api/analysis/query` | 条件查询（占位） |
| GET | `/api/analysis/export` | 导出（仅 admin，开发中） |

**均需登录。**

### 4.1 历史场次列表

`GET /api/replay/runs?page=1&size=50`

**响应**

```json
{
  "success": true,
  "runs": [
    {
      "id": 29,

```

### 4.2 按 ID 获取回放数据

`GET /api/replay/runs/{runId}`

**响应**

```json
{
  "success": true,
  "data": {
    "type": "REPLAY_DATA",
    "mapWidth": 30,
    "mapHeight": 30,
    "maxTick": 420,
    "explorationRate": 100,
    "taskConfig": { "mapWidth": "30", "algorithm": "BFS", ... },

```

- `carHistories`：每车 **实际轨迹**（`x,y,tick`），非规划 `RouteList`

## 6. WebSocket（`:8888`）

实现：`display/.../WebSocketBridge.java`  
连接 URL：`ws://<displayHost>:8888`（与 HTTP 同主机）

### 6.1 浏览器 → 服务端（命令）

除 `ADD_CAR`、`REQUEST_REPLAY` 在 Display 本地处理外，其余命令 **原样转发** 到 RabbitMQ `ControllerCmd`。

#### 开始仿真

```json
{
  "type": "SET_CONFIG",
  "data": {
    "mapWidth": "30",
    "mapHeight": "30",
    "carCount": "3",
    "obstacleRatio": "0.15",
    "algorithm": "BFS",
    "tickInterval": "500",
    "active": "true",
    "operator": "admin"
  }
}
```

#### 重置

```json
{ "type": "RESET", "data": {} }
```

#### 暂停 / 继续

```json
{ "type": "TOGGLE_PAUSE" }
```

#### 调整节拍间隔

```json
{ "type": "SET_TICK_INTERVAL", "data": { "interval": 500 } }
```

- `interval` 范围：100～2000（毫秒）

#### 切换障碍格（右键）

```json
{ "type": "TOGGLE_OBSTACLE", "data": { "row": 5, "col": 10 } }
```

#### 动态添加小车

```json
{ "type": "ADD_CAR" }
```

由 Display 在 **本机** 启动 `car` JAR 进程（`--dynamic`），不经过 Controller 转发。

#### 请求回放（当前场次）

```json
{ "type": "REQUEST_REPLAY" }
```

可选带 `runId`（一般由 HTTP 拉历史场次，WS 主要用于当场）。

### 6.2 服务端 → 浏览器（推送）

#### 实时仿真状态

Controller 发 `REFRESH_ALL` 后，Display 读 Redis 组装 JSON 广播，字段与 `SimulationState` 类似：

```json
{
  "tick": 42,
  "explorationRate": 67,
  "taskConfig": { "mapWidth": "30", ... },
  "runStartedBy": "admin",
  "cars": [
    {
      "carId": "Car001",
      "number": 1,
      "position": { "x": 5, "y": 8 },
      "target": { "x": 12, "y": 3 },
      "routeList": [ { "x": 6, "y": 8 } ],
      "status": "READY",
      "steps": 40,
      "effectiveSteps": 28
    }
  ],
  "mapView": [[false, true, ...]],
  "mapBlock": [[...]],
  "mapSealed": [[...]]
}
```

大地图时使用 `mapViewB64` / `mapBlockB64` / `mapSealedB64` 替代二维数组。

#### 回放数据

`type: "REPLAY_DATA"`，结构见 §4.2。

#### 加车事件

| type | 说明 |
|------|------|
| `CAR_PENDING` | 正在启动，含 `carId` |
| `CAR_LAUNCHED` | 进程就绪 |
| `CAR_LAUNCH_FAILED` | 失败原因 `reason` |

#### 错误

```json
{ "type": "REPLAY_ERROR", "error": "场次不存在" }
```

### 6.3 刷新页面 vs 断线重连

| 场景 | 行为 |
|------|------|
| **F5 整页刷新** | 前端状态清空；**不**自动 RESET；后台仿真可能仍在跑 |
| **WebSocket 断线重连（未刷新）** | 自动发 `RESET`，清空仿真黑板 |

## 7. MQ 消息（模块间）

类型常量：`common/.../mq/MessageTypes.java`  
消息格式：`common/.../mq/MessageBuilder.java`

### 7.1 统一 JSON 结构

```json
{
  "type": "PLAN_ROUTE",
  "tick": 12,
  "carId": "Car001",
  "timestamp": 1710000000000,
  "data": { }
}
```

- `carId`：单车消息必填；系统消息可省略
- `data`：无数据时为 `{}`

### 7.2 队列

| 队列名 | 消费者 | 说明 |
|--------|--------|------|
| `ControllerCmd` | Controller（**仅 1 个**） | 各模块回执 + 前端命令 |
| `TaskConfigCmd` | TaskConfigurator（**仅 1 个**） | 初始化 / 重置 |
| `TargetPlannerCmd` | TargetPlanner | 可分多实例竞争消费 |
| `NavigatorCmd` | Navigator | 可分多实例竞争消费 |
| `StrategySupervisorCmd` | StrategySupervisor | 建议 1 个 |
| `Car_{carId}` | 对应 Car 进程 | 如 `Car_Car001` |
| `UpdateView`（Fanout） | Display 等 | `REFRESH_ALL` 广播 |

### 7.3 消息类型一览

| type | 发布者 → 订阅者 | 说明 |
|------|-----------------|------|
| `SET_CONFIG` | Display → Controller | 用户点「开始」 |
| `FORWARD_CONFIG` | Controller → TaskConfigurator | 转发配置 |
| `TASK_READY` | TaskConfigurator → Controller | 黑板初始化完成 |
| `RESET` | Display → Controller | 用户点「重置」 |
| `FORWARD_RESET` | Controller → TaskConfigurator | 转发重置 |
| `ASSIGN_TARGET` | Controller → TargetPlanner | 分配目标 |
| `TARGET_ASSIGNED` | TargetPlanner → Controller | 目标已写入黑板 |
| `PLAN_ROUTE` | Controller → Navigator | 请求算路 |
| `ROUTE_PLANNED` | Navigator → Controller | 路线已写入黑板 |
| `SUPERVISE_ROUTE` | Controller → StrategySupervisor | 路线监督 |
| `ROUTE_OPTIMIZED` | StrategySupervisor → Controller | 监督完成 / 换线 / 重分配 |
| `TICK_MOVE` | Controller → Car | **唯一**移动节拍指令 |
| `MOVED` | Car → Controller | 移动一步完成 |
| `ROUTE_DONE` | Car → Controller | 当前路段走完 |
| `BLOCKED` | Car → Controller | 下一步受阻 |
| `BLOCKED_TIMEOUT` | Controller → Car | 阻塞超时，清路线 |
| `REFRESH_ALL` | Controller → Display（Fanout） | 触发界面刷新 |
| `TOGGLE_PAUSE` | Display → Controller | 暂停/继续 |
| `SET_TICK_INTERVAL` | Display → Controller | 调速 |

### 7.4 典型 `data` 字段

**`PLAN_ROUTE`**

```json
{
  "carId": "Car001",
  "subStart": { "x": 1, "y": 2 },
  "subTarget": { "x": 10, "y": 8 },
  "algorithm": "BFS",
  "supervised": false
}
```

**`ROUTE_PLANNED`**

```json
{ "routeFound": true, "routeLength": 15 }
```

**`ROUTE_OPTIMIZED`**

```json
{
  "optimized": true,
  "overlapReassign": false,
  "oldLength": 12,
  "newLength": 10
}
```

- `overlapReassign: true` → Controller 清目标/路线，车回 `IDLE` 重分

**`MOVED`**

```json
{
  "newPosition": { "x": 6, "y": 8 },
  "routeRemaining": 5
}
```

**`REFRESH_ALL`**

```json
{ "explorationRate": 67 }
```

## 8. Redis 黑板（非 HTTP，供联调对照）

实现：`common/.../redis/BlackboardClient.java`

| Key / 模式 | 内容 |
|------------|------|
| `mapView` | 已探索位图 |
| `mapBlock` | 障碍位图 |
| `mapSealed` | 密封区（不可达） |
| `mapHeat` | 热力（经过次数） |
| `TaskConfig` | 任务配置 Hash |
| `Car*:Status` | 车五态 |
| `Car*:Position` | 车坐标 |
| `Car*:Target` | 目标格 |
| `Car*:RouteList` | **当前规划路线**（不落 SQL） |
| `Car*:History` | **实际轨迹** `{"x","y","tick"}`（回放归档用） |
| `explorationEvents` | 首次探索事件 `tick,row,col` |
| `controller:instance` | Controller 单实例锁 |
| `auth:session:*` | 登录 Session |

车数量：**不写在消息里**，各模块 `discoverCarIds()` 扫描 `Car*:Status`。

## 11. 联调检查清单

1. Redis / RabbitMQ 已启动，各模块 `infra.local.json` 指向同一 `redisHost` / `mqHost`
2. RabbitMQ 管理台：`ControllerCmd`、`TaskConfigCmd` **Consumers = 1**
3. 浏览器：`http://<displayHost>:8887` 登录后打开仿真页
4. 点「开始」后应看到 `TASK_READY`，车开始调度
5. 监督阶段需 `StrategySupervisor` 在线，否则可能卡在 `awaitingSupervision`

## 问题 1：障碍物有时突然消失

**原因**：不是 bug，是 `DynamicObstacleUtil` 在每 20 tick 随机移除 ≤2 个障碍物。

**涉及文件**

`controller/src/main/java/com/substation/controller/StatusDispatcher.java`

**当前代码**（dispatch 方法中）

```java
tick++;

if (tick % DYNAMIC_OBSTACLE_INTERVAL == 0) {
    Set<Point> carPositions = new HashSet<>();
    for (String cid : bb.discoverCarIds()) {
        bb.getCarPosition(cid).ifPresent(carPositions::add);
    }
    int mapW = Math.max(bb.getMapWidth(), 30);
    int mapH = Math.max(bb.getMapHeight(), 30);
    List<String> changes = dynamicObstacleUtil.generate(bb, mapW, mapH, carPositions);
    if (!changes.isEmpty()) {
        System.out.println("[Controller] 动态障碍: " + String.join(" ", changes));
    }
}

if (bb.getExplorationRate() >= EXPLORATION_COMPLETE) {
```

**修改方案**

删除整个 `if (tick % DYNAMIC_OBSTACLE_INTERVAL == 0)` 代码块，并清理相关 import 和字段：

- 删除 `import com.substation.common.DynamicObstacleUtil;`
- 删除 `import java.util.HashSet;`、`import java.util.List;`（如仅用于此功能）
- 删除字段 `DYNAMIC_OBSTACLE_INTERVAL`
- 删除字段 `dynamicObstacleUtil`

## 问题 2：地图全探索完也没有停止

**原因**：`DynamicObstacleUtil` 移除障碍物导致可探索格数（分母）变大，探索率反而下降，永远达不到 99%，无法触发 `completeTask()`。

探索率公式：`已探索格数 × 100 ÷ (总格数 - 障碍物数)`。随动态移除累积，分子增长率不及分母增大速度。

**修复**：问题 1 删除 `DynamicObstacleUtil` 后，障碍物不再变化，探索率单调递增，此问题自动解决。无需额外修改。

## 问题 3：小车卡住不动，暂停再开始又恢复

**原因**：这是 Controller 的超时恢复机制在工作，不是 bug。

| 场景 | 机制 | 代码位置 |
|------|------|----------|
| Car 设 MOVING 后崩溃卡住 | 连续 ≥2 tick 仍 MOVING → Controller 强制切 READY | `checkMovingStuck()` |
| 下一步被障碍物挡住 | Car 清 RouteList → 设 BLOCKED → ≥2 tick 后 Controller 清 Target/Route → 设 IDLE → 下轮重新分配 | `checkBlockedTimeout()` |

暂停期间消息堆积，恢复后集中处理，所以看起来"又好了"。如果经常触发，说明 TargetPlanner 的贪心分配在某些场景下找不到合适目标（距离 ≥10 规则在剩余格子少时失效）。如果需要缓解，可把 `GreedyTargetAllocator` 的 `MIN_TARGET_DISTANCE`（当前 10）调小或加回退逻辑。

**无需修改** Controller 本身，恢复机制正常运作。可选优化在 Person C 的 TargetPlanner 中。

## 问题 4：tick 调速滑块拖不动

**原因**：前端 JS 的问题。后端 `TickScheduler.setInterval()` 逻辑正确，前端滑块可能事件未绑定、消息未发送或 WebSocket 未就绪。

**涉及文件**

`display/src/main/resources/web/js/app.js`

**排查要点**

1. 滑块 HTML 的 `min`/`max`/`step` 属性是否设置：`<input type="range" min="100" max="2000" step="100" value="500">`
2. `addEventListener('input', ...)` 是否绑定
3. 回调中 `ws.readyState === WebSocket.OPEN` 是否成立
4. 消息格式中 `data.interval` 字段是否为数字类型

**修改方案**

```javascript
const slider = document.getElementById('tickIntervalSlider');
slider.addEventListener('input', () => {
    if (ws && ws.readyState === WebSocket.OPEN) {
        ws.send(JSON.stringify({
            type: 'SET_TICK_INTERVAL',
            data: { interval: Number(slider.value) }
        }));
    }
});
```

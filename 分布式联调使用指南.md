# 分布式联调使用指南（每人一份）

> **适用场景**：四人四机通过 Tailscale 联调变电站巡检仿真。  
> **核心思路**：每人本机维护一份 `deploy/infra.local.json`。  
> **IP 只配置一次**（写入文件）；之后每次联调只跑 `start-*.ps1`，**不用**再设 `$IP_A` / `$REMOTE`，也不用给每条 `mvnw` 命令加参数。

### 配置文件到底省了什么？

| 以前（无配置文件） | 现在（有配置文件） |
|------------------|-------------------|
| 每次开 PowerShell 都要 `$IP_A = "..."` | **不用** |
| 每个模块命令都要 `-Dexec.args="--redis-host ..."` | **不用**，Java 自动读文件 |
| 关窗口重开、第二天再联调，重新敲一遍 | 直接 `start-cars.ps1` 等 |
| 车数、Display 地址散落在各处 | 全在 `infra.local.json` 里改一处 |

`setup-config.ps1` 只在**第一次**（或 IP 变更时）运行；B/C/D 也可直接手改 `deploy/infra.local.json`，连 setup 都不用跑。

---

## 1. 分工速查

| 人员 | 负责什么 | 本机要起的进程 |
|------|----------|----------------|
| **Person A** | 基础设施 + 调度 | Docker（Redis + MQ）、**Controller（必须最后起）** |
| **Person B** | 小车 | Car001、Car002、Car003（可在配置里增减） |
| **Person C** | 规划知识源 | TaskConfigurator、Navigator、TargetPlanner、StrategySupervisor |
| **Person D** | 展示 | SQL Server、Display、浏览器点「开始」 |

**全组硬约束**

- 只有 **一套** Redis + RabbitMQ（在 Person A 的 Docker 里）
- 只有 **一个** Controller、**一个** Display
- 同一辆车（如 `Car001`）不能在两台电脑各起一个进程

---

## 2. 每人第一次要做什么（四台都一样）

### 2.1 安装与编译

```powershell
cd D:\car_homework
.\mvnw.cmd install -DskipTests
```

### 2.2 确认 Tailscale 互通

```powershell
tailscale status
tailscale ip -4
```

Person A 把本机 Tailscale IP 发到群里（下文记为 **IP_A**），其他人互 ping 一下：

```powershell
ping IP_A
```

### 2.3 生成本机配置文件（只做一次）

每人按自己的角色执行 **一条** `setup-config` 命令（见下一节各角色说明）。  
会在项目里生成 `deploy/infra.local.json`（已 gitignore，不会把个人 IP 提交到 Git）。

之后联调，只需运行对应的 `start-*.ps1` 脚本即可。

---

## 3. Person A（基础设施 + Controller）

### 3.1 你是谁

Docker 跑 Redis 和 RabbitMQ 的那台机器；**全组唯一**起 Controller 的人；Controller **必须最后起**。

### 3.2 首次配置

```powershell
cd D:\car_homework
.\scripts\setup-config.ps1 -Role infra
```

会生成 `deploy/infra.local.json`，其中 `redisHost` / `mqHost` 为 `localhost`。

### 3.3 每次联调

**第一步：起 Docker**

```powershell
cd D:\car_homework
.\scripts\start-infra.ps1
```

（脚本会执行 `docker compose up -d` 并打印当前配置。）

**第二步：防火墙（管理员 PowerShell，本机只做一次）**

```powershell
New-NetFirewallRule -DisplayName "Redis6379" -Direction Inbound -Protocol TCP -LocalPort 6379 -Action Allow
New-NetFirewallRule -DisplayName "RabbitMQ5672" -Direction Inbound -Protocol TCP -LocalPort 5672 -Action Allow
New-NetFirewallRule -DisplayName "RabbitMQ15672" -Direction Inbound -Protocol TCP -LocalPort 15672 -Action Allow
```

**第三步：把 IP 发到群里**

```powershell
tailscale ip -4
```

告知 B/C/D：**基础设施 IP 是 xxx**。

**第四步：等 B、C、D 在群里说「已就绪」**

**第五步：最后起 Controller**

```powershell
.\scripts\start-controller.ps1
```

成功日志应出现：`[Controller] 控制器已启动，等待任务配置...`

### 3.4 你不需要做的事

- 不用改 `redisHost` 为 Tailscale IP（本机 Docker 用 `localhost` 即可）
- 不要在 B/C/D 就绪之前起 Controller

---

## 4. Person B（小车）

### 4.1 你是谁

负责跑 **Car 进程**的人；车跑在你这台电脑上，但数据写在 Person A 的 Redis 里。

### 4.2 首次配置

把 `100.94.124.1` 换成 Person A 发来的 **IP_A**（脚本会提示输入，**只需这一次**）：

```powershell
cd D:\car_homework
.\scripts\setup-config.ps1 -Role car
```

### 4.3 每次联调

```powershell
cd D:\car_homework
.\scripts\start-cars.ps1
```

会按 `deploy/infra.local.json` 里 `cars` 数组逐个开窗口，默认 3 辆：`Car001`、`Car002`、`Car003`。

群里回复：**「小车已就绪」**。

### 4.4 调整车辆数量

编辑 `deploy/infra.local.json` 的 `cars` 字段，例如 5 辆车：

```json
"cars": ["Car001", "Car002", "Car003", "Car004", "Car005"]
```

保存后重新运行 `.\scripts\start-cars.ps1`。

### 4.5 检查是否连对

每个 Car 窗口启动日志里应看到连向 **IP_A:6379**，而不是 `localhost`。  
本机 **不要** 自己 `docker compose` 起 Redis（会和全组共用黑板冲突）。

---

## 5. Person C（规划模块）

### 5.1 你是谁

负责 TaskConfigurator、Navigator、TargetPlanner、StrategySupervisor；**不负责** Car（那是 Person B）。

### 5.2 首次配置

```powershell
cd D:\car_homework
.\scripts\setup-config.ps1 -Role planner
```

### 5.3 每次联调

```powershell
cd D:\car_homework
.\scripts\start-planner.ps1
```

会打开 **4** 个 cmd 窗口。群里回复：**「规划模块已就绪」**。

### 5.4 扩展 Navigator 算力（可选）

若想在**本机或其他同学电脑**多开 Navigator（多实例竞争消费 `NavigatorCmd` 队列）：

```powershell
.\scripts\start-navigator.ps1 -Count 2
```

无需改 Java 代码；确保该机器也有正确的 `deploy/infra.local.json`（`redisHost` / `mqHost` 指向 IP_A）。

---

## 6. Person D（Display + 操作浏览器）

### 6.1 你是谁

跑 Display 和 SQL Server；**全组唯一**在浏览器里点「开始」的人（避免重复下发配置）。

### 6.2 首次配置

```powershell
cd D:\car_homework
.\scripts\setup-config.ps1 -Role display
```

脚本会提示输入 Person A 的 IP，并尝试自动识别本机 Tailscale IP 作为 `displayHost`。

### 6.3 每次联调

**先确保 SQL Server 已启动**（登录依赖它）。

```powershell
cd D:\car_homework
.\scripts\start-display.ps1
```

群里回复：**「Display 已就绪」**。

**Person A 起好 Controller 之后**，在本机浏览器打开：

```text
http://localhost:8887
```

登录 → 进入仿真页 → **点「开始」**。

### 6.4 让别人同步观看

Person D 本机用 `http://localhost:8887` 点「开始」；其他人访问 `http://<D的TailscaleIP>:8887` 即可实时看同一画面。  
详见 **`多人同步观看网页教程.md`**（D 机防火墙放行 8887/8888）。

## 7. 全组启动顺序（记这一张图即可）

```
Person A   setup-config → start-infra.ps1（Docker）
Person C   setup-config → start-planner.ps1
Person B   setup-config → start-cars.ps1      ← 可与 C 并行
Person D   setup-config → start-display.ps1
Person A   等 B/C/D 就绪 → start-controller.ps1   ← 必须最后
Person D   浏览器点「开始」
```

## 8. 配置文件说明（`deploy/infra.local.json`）

| 字段 | 含义 | Person A | Person B/C/D |
|------|------|----------|--------------|
| `redisHost` | Redis 地址 | `localhost` | Person A 的 **IP_A** |
| `mqHost` | RabbitMQ 地址 | `localhost` | **IP_A** |
| `redisPort` / `mqPort` | 端口 | 默认 6379 / 5672 | 一般不用改 |
| `role` | 角色标记 | `infra` | `car` / `planner` / `display` |
| `displayHost` | Display 机器 IP | `localhost` | D 的本机 Tailscale IP |
| `cars` | 小车 ID 列表 | 可忽略 | **Person B** 按此启动 |

**优先级**：命令行 `--redis-host` 等 **覆盖** 配置文件；配置文件 **覆盖** 默认 localhost。

也可手动编辑 JSON，不必重新跑 `setup-config`。

环境变量 `CAR_HOMEWORK_CONFIG` 可指向其他路径的配置文件（高级用法）。

## 9. 脚本一览

| 脚本 | 谁用 |
|------|------|
| `scripts/setup-config.ps1` | 四人首次各跑一次（参数不同） |
| `scripts/start-infra.ps1` | Person A |
| `scripts/start-controller.ps1` | Person A（最后） |
| `scripts/start-planner.ps1` | Person C |
| `scripts/start-cars.ps1` | Person B |
| `scripts/start-display.ps1` | Person D |
| `scripts/start-navigator.ps1` | 任意（扩算力） |

`setup-config.ps1` 常用参数（均可省略，按提示交互输入）：

```text
-Role infra|planner|car|display     # B/C/D 会自动按远程模式处理
-InfraHost <IP_A>                   # 可选，不填则运行时提示输入一次
-DisplayHost <IP>                   # 可选，display 角色会自动检测 Tailscale IP
```

## 10. 联调检查（30 秒自检）

| 检查项 | 命令 / 现象 |
|--------|-------------|
| 能 ping 通 A | `ping IP_A` |
| 远程 Redis 通 | `Test-NetConnection IP_A -Port 6379` |
| A 的 Docker | `docker ps` 有 redis、rabbitmq |
| B 的车 | 3 个 Car 窗口，日志为 `IP_A:6379` |
| C 的模块 | 4 个窗口无报错 |
| D 的页面 | `http://localhost:8887` 能打开 |
| Controller 唯一 | 仅 A 一台，且是最后起的 |
| 仿真跑起来 | D 点开始后小车动、探索率涨 |

## 11. 常见问题

| 现象 | 处理 |
|------|------|
| 脚本提示缺少 `infra.local.json` | 先运行 `.\scripts\setup-config.ps1 ...` |
| 连不上 Redis | 检查 IP_A、A 的防火墙、A 的 Docker |
| Controller 秒退 | 全组只保留 A 上一个 Controller |
| 车不动 | 确认 A **最后**起了 Controller，且 B 的车已起 |
| 登录失败 | D 本机 SQL Server 未启动 |
| 联调后单机跑不起来 | `.\scripts\setup-config.ps1 -Role infra`，再 `docker compose up -d` 和 `.\start_all.bat` |

---

## 12. 联调结束后恢复单机

在 **B、C、D** 本机：

```powershell
cd D:\car_homework
.\scripts\setup-config.ps1 -Role infra
docker compose up -d
.\start_all.bat
```

Person A 若也要单机跑全套，同样保证 `infra.local.json` 里是 `localhost`，然后 `.\start_all.bat`。
